  `docs/api/auth.md` promises "explicit Tavern-issued credentials" for
  external clients; after 003 the natural next step is a pairing UX (token

  `specs/agent-runtimes/`. The data model (participants, per-agent sessions)

  `DISCORD_BOT_TOKEN` / `DISCORD_APPLICATION_ID` in `.env.example` ("Runtime

  Discord presentation logic in `apps/server/src/chat/list.ts`, and the dead
  messenger package (removed by plan 005). Either finish the channel story or
  excise the presentation stubs; the half-state costs maintenance on every
  chat-list change.
