## Why this matters

Two `packages/` directories are dead weight: `packages/honcho-graph`
(~856 LoC of TypeScript with zero imports anywhere in the monorepo) and
`packages/tavern-openclaw-messenger` (contains only a stray `src/actions.js`,
no `package.json` — not even a valid workspace member). Dead workspaces slow
dependency resolution, confuse contributors and agents ("is this for future
work?"), and show up in searches. The repo's own guidance (AGENTS.md) prefers
clean breaks over carrying remnants.

## Current state

- `packages/tavern-openclaw-messenger/` — contains only `src/actions.js`
  (~7.7 KB). No `package.json`, so bun does not treat it as a workspace.
  Likely a leftover from a renamed/removed adapter package (git history references
  `packages/openclaw-gateway-adapter`, also gone).
- `packages/honcho-graph/` — a full package (~856 LoC in `src/`). Verified at
  planning time: `grep -rln "honcho" apps packages jobs scripts` (excluding
  the package itself and node_modules) returns nothing.
- Root `package.json` workspaces are globbed (`"packages/*"`), so no
  workspace-list edit is needed. The root `build` and `typecheck` script
  chains do NOT include either package (they list api, sdk, claude-usage,
  codex-usage, server, runtime, website only).
- `bun.lock` will shrink when a workspace disappears — regenerating it via
  install is part of this plan.
