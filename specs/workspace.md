# Workspace

Tavern owns the managed OpenClaw workspace it creates for Tavern agents.

The workspace is the prompt-facing home that OpenClaw loads for each agent
session. Tavern renders OpenClaw-native bootstrap files for managed agents:
`AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, and `USER.md`.

## Instruction Sources

The workspace bundle starts from two Tavern-owned sources:

* **Seed defaults.** Repo-managed lower-case markdown defaults explain baseline
  operating doctrine, identity, tool routing, and user context for a new managed
  agent.
* **Workspace files.** `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, and
  `USER.md` live in the managed OpenClaw workspace and become the editable
  source of truth after seeding.

The rendered files should read as natural OpenClaw bootstrap documents. Section
boundaries are ownership concerns; they should not make the prompt feel like a
generated template.

The repo defaults live under `apps/runtime/src/workspace/seed-defaults/`.
Runtime renders those lower-case seed files to OpenClaw's uppercase workspace
file names. Runtime substitutes narrow placeholders such as Cortex page types
and agent name while seeding.

## Runtime Behavior

Runtime seeds missing bootstrap files in the managed OpenClaw workspace on boot
and config sync. Existing workspace files are left intact so agents and users
can edit `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, and `USER.md`
directly. Runtime replaces only missing files or old Tavern-generated files that
still carry the legacy "generated by Tavern" or workspace-notes marker.

Runtime leaves optional bootstrap files such as `HEARTBEAT.md`, `BOOTSTRAP.md`,
`MEMORY.md`, and `ROLE.md` present but blank. Blank files avoid missing-file
markers without enabling heartbeat, bootstrap, or extra memory behavior Tavern
does not own.

Workspace files are the source of truth after seeding. If an agent needs to
remember durable operating guidance about itself, it edits the appropriate
workspace markdown file. Cortex remains the place for durable facts, sources,
people, projects, and decisions.

## App Surfaces

The Tavern app exposes each workspace markdown file as its own settings page.
Edits go through the Runtime agent file API and update the live workspace file.

## Tavern Workspace Plugin

The first-party OpenClaw plugin `tavern-workspace` is reserved for future
workspace integration. It does not expose agent-notes tools and does not block
workspace file edits.

It does not own chat delivery, Cortex recall/capture/search, skills, model
settings, OpenClaw runtime config, or workspace-file write policy.
